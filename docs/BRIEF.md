# Landing Page para el encuentro de egresados — The Last INALFU

---

## 1. Título de la tarea

Diseño y desarrollo de una **landing page conmemorativa para el encuentro de egresados “The Last INALFU — 20 años después”**, construida sobre la estructura técnica existente del proyecto base, usando **HTML, JavaScript Vanilla y TailwindCSS**, con una experiencia visual **nostálgica, elegante, cercana y memorable**.

---

## 2. Contexto

Se necesita transformar una landing page originalmente diseñada para una veterinaria en una nueva experiencia digital para convocar, emocionar e informar sobre el encuentro de egresados **“The Last INALFU — 20 años después”**.

La página debe funcionar como una **pieza de convocatoria y memoria colectiva**. No solo debe comunicar claramente el evento, sino también transmitir el espíritu de una generación que compartió una etapa importante de crecimiento, parche, exploración juvenil y recuerdos del colegio.

El sitio no está dirigido únicamente a quienes se graduaron formalmente con la promoción, sino también a quienes estudiaron con ella, compartieron su contexto, parcharon con el grupo o hicieron parte de esa época. La idea es que la página se sienta como una invitación abierta a todos los que vivieron ese tiempo.

El tono debe mezclar:
- nostalgia
- alegría del reencuentro
- humor y guiños internos del parche
- cuidado visual
- calidez humana

La página será desarrollada como un proyecto de **Guineo Lab**, aunque esta firma tendrá una presencia **discreta al final del sitio**.

Además de convocar, la landing debe comenzar a consolidarse como un posible **archivo vivo del encuentro**, con potencial futuro para seguir alojando fotos, recuerdos y materiales después del evento.

---

## 3. Objetivo del proyecto

La landing debe cumplir cuatro funciones principales:

1. **Convocar al encuentro**
2. **Centralizar la información principal del evento**
3. **Despertar emoción, nostalgia e identificación**
4. **Invitar a la interacción futura** con recuerdos, anécdotas, emprendimientos y dinámicas simples del parche

La acción principal del usuario será:
- informarse sobre el evento
- sentirse invitado
- entrar al grupo de WhatsApp del encuentro

CTA principal sugerido:
**“Ser parte del reencuentro”**

---

## 4. Público objetivo

La página está dirigida a:
- quienes se graduaron con la promoción
- quienes estudiaron con la promoción y no necesariamente se graduaron
- quienes parcharon con el grupo
- quienes hicieron parte de esa etapa de juventud y adolescencia
- quienes compartieron el contexto humano, social y emocional de esos años

Mensaje clave de inclusión:
**“Si hiciste parte de esa época, este encuentro es para ti.”**

---

## 5. Requerimientos técnicos

### Stack
- HTML5
- JavaScript Vanilla
- TailwindCSS

### Enfoque técnico
La landing debe **mantener la estructura técnica del proyecto existente** para facilitar su adaptación.

La idea no es rehacer el proyecto desde cero, sino **reutilizar el repositorio base**, conservando:
- estructura general
- layout principal
- navegación
- secciones
- responsive base
- JavaScript simple existente

Se podrán modificar:
- textos
- nombres de secciones
- colores
- imágenes
- iconografía
- detalles visuales
- tamaños y diagramación si hace falta, sin alterar radicalmente el layout

---

## 6. Objetivo funcional

La landing debe presentar de forma clara y emocional la propuesta del encuentro, guiar al usuario a unirse al grupo de WhatsApp e introducir futuras posibilidades de participación como:
- envío de anécdotas
- envío de audios
- publicación de emprendimientos
- interacción lúdica/nostálgica tipo “crush perfecto”

En esta **versión 1**, la prioridad funcional será:
- informar
- convocar
- conectar al grupo de WhatsApp

---

## 7. Estructura mínima requerida

La página será **una sola landing page con scroll**.

### Secciones requeridas

#### 1. Hero principal
Debe incluir:
- nombre del evento: **The Last INALFU**
- subtítulo: **20 años después**
- una frase de invitación
- CTA principal: **Ser parte del reencuentro**
- imagen principal del colegio

#### 2. Sobre el encuentro
Texto emocional y conmemorativo sobre:
- lo que significó esa época
- el paso del tiempo
- el valor de volver a encontrarse
- el espíritu del parche

Aquí puede ir el texto más narrativo y sensible.

#### 3. Quiénes están invitados
Sección más clara y directa para explicar que:
- no es solo para graduados
- también es para quienes compartieron esa etapa
- si viviste esa época, este encuentro también es tuyo

#### 4. Galería de fotos
Galería sencilla con aproximadamente 5 fotos:
- colegio
- fiesta/carnaval
- fotos antiguas del grupo
- imágenes con valor evocador

#### 5. Información del evento
Debe incluir:
- fecha: **26 de diciembre de 2026**
- reservar el día
- encuentro después del final de la tarde
- cualquier otra información que luego se agregue

#### 6. Anécdotas / recuerdos
En V1 puede ser una sección de contenido e invitación.
Más adelante puede enlazarse a un formulario o canal externo para recopilar:
- recuerdos escritos
- anécdotas
- audios

#### 7. Emprendimientos de egresados
Dejar espacio para al menos **10 emprendimientos**, con estructura tipo tarjeta:
- nombre de la persona
- nombre del emprendimiento
- descripción corta
- enlace o contacto

En V1 pueden quedar placeholders o estructura preparada.

#### 8. Acceso al grupo de WhatsApp
Sección clara con CTA al grupo oficial del encuentro:

`https://chat.whatsapp.com/H2MBfKG3SPA5kyL436iNJP`

#### 9. Footer
Debe incluir:
- cierre emocional breve
- referencia al encuentro
- firma discreta de **Guineo Lab**

---

## 8. Posible funcionalidad futura

Estas ideas no son obligatorias en V1, pero deben quedar contempladas como evolución futura del proyecto:

- formulario de anécdotas en texto
- envío de notas de voz
- formulario de emprendimientos
- archivo fotográfico post-encuentro
- preservación de la página como memoria digital del encuentro
- dinámica lúdica de “crush perfecto” a partir de nombres de hombres y mujeres de la promoción

### Sobre esta última idea
Sí es posible hacerla de forma simple en JavaScript:
- lista local de nombres
- campo de entrada
- lógica básica aleatoria o semi-dirigida
- resultado divertido tipo “tu crush del 2006 habría sido…”

Eso puede ir perfecto como una **V2 o mini módulo especial**, pero no es necesario meterlo de entrada.

## 9. Estilo visual esperado

La dirección visual debe conservar una sensación:
- institucional refinada
- nostálgica
- cálida
- cercana
- juvenil con madurez
- emotiva, pero prolija

### Referencias visuales
- colores del colegio como base
- presencia del verde institucional y blanco
- posibilidad de inspirarse sutilmente en formas o símbolos asociados al colegio y al escudo
- guiños suaves al estilo de Guineo Lab
- detalles manuales o emocionales si se integran con criterio

### Estilo esperado
- diseño limpio
- apariencia premium y cuidada
- visualmente nostálgico pero no viejo
- elegante sin rigidez
- con algunos detalles emocionales/manuales
- uso estratégico de espacios en blanco
- jerarquía visual clara
- responsive desde móvil

## 13. Constraints (Restricciones)

- Usar únicamente **HTML, TailwindCSS y JavaScript Vanilla**
- No usar frameworks como React, Vue o Angular
- No cambiar radicalmente la arquitectura base del proyecto existente
- Mantener el layout general del template original, adaptándolo con criterio
- No usar librerías externas complejas
- El código debe mantenerse semántico y ordenado
- La landing debe ser responsive en móvil, tablet y desktop
- Debe priorizarse el rendimiento
- La interfaz debe ser accesible:
  - contraste legible
  - textos comprensibles
  - botones visibles
  - estructura clara
- El diseño debe evitar verse:
  - infantil
  - recargado
  - genérico
  - excesivamente institucional
- La experiencia debe sentirse:
  - nostálgica
  - humana
  - elegante
  - cálida
  - con humor sutil del parche

## 14. Mapping sugerido desde el template veterinario

Para facilitar la adaptación, estas serían las equivalencias recomendadas:

- **Hero veterinaria** → **Hero del encuentro**
- **Servicios** → **Momentos / secciones del encuentro / recuerdos**
- **Beneficios o diferenciadores** → **Quiénes están invitados / por qué este encuentro importa**
- **Testimonios** → **Anécdotas / frases del parche / recuerdos**
- **Contacto / CTA final** → **Únete al grupo de WhatsApp**
- **Footer** → **Cierre conmemorativo + firma discreta de Guineo Lab**

Esto permite mantener la estructura y cambiar principalmente:
- copy
- imágenes
- intención
- identidad visual

- Existe una landing page funcional usando la estructura actual del repositorio:
  - `index.html`
  - TailwindCSS
  - `script.js`
- La página incluye estas secciones:
  - hero
  - sobre el encuentro
  - quiénes están invitados
  - galería
  - información del evento
  - anécdotas / recuerdos
  - emprendimientos
  - acceso a WhatsApp
  - footer

- El CTA al grupo de WhatsApp es visible y claro

- La experiencia visual transmite:
  - nostalgia
  - alegría
  - pertenencia
  - cuidado
- El resultado no parece una plantilla veterinaria reciclada, sino una página pensada para esta historia y esta comunidad
